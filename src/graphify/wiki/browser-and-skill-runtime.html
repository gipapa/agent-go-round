Browser and Skill Runtime

在 AgentGoRound 裡，skill 不只是 prompt snippet，而是一個 workflow layer。特別是 browser workflow skill， 它把 observation、planning、tool execution、completion gate 都做成可追蹤的多輪 runtime。

核心循環

  • observe：先對頁面狀態與 browser snapshot 做結構化整理。
  • act：再決定要 open、click、fill、wait 還是 request user help。
  • sync state：把 tool output 與 browser observation 合併回 runtime。
  • completion gate：只有在足夠 grounded 時才會 finish。

這條能力線解決了什麼

  • 把「看起來像 agent 做到了」和「真的走完 browser workflow」區分開來。
  • 讓 blocked / manual gate / timeout / observe-first 這些情境能在 UI 被清楚顯示。
  • 讓 case 6 這種 GitHub Trending 多輪案例可以反覆驗證與回歸測試。
