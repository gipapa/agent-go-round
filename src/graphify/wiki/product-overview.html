Product Overview

AgentGoRound 的核心不是「幫你包裝一個固定 agent 產品」，而是提供一個在瀏覽器中就能直接管理 agent workflow 的 playground。 因為部署目標是 GitHub Pages，所以整體設計天然偏向 frontend-only 與 local-first。

產品主張

  • 在同一個 workspace 中管理 agents、docs、MCP tools、built-in tools、skills 與 chat history。
  • 重點是快速驗證 use case、做教學演示、研究 agentic UI，而不是提供穩定後端 SaaS。
  • 多數狀態都放在瀏覽器：`localStorage`、`IndexedDB`、以及目前頁面環境本身。
這也是為什麼 README 會一直重複強調 browser-first。對這個專案來說，那不是部署細節，而是產品邊界。

主要能力線

  • Agent Workspace：組織所有對話與能力授權的中心。
  • Load Balancer + Credential Pool：決定模型、key 與 failover。
  • Docs / MCP / Built-in Tools / Skills：四種主要能力來源。
  • Tutorials：把功能串成可操作、可驗證的 journey。
  • MAGI：多 agent 裁決模式。
