Agent and Routing

AgentGoRound 的 agent 不直接綁 provider，而是綁到 Load Balancer。這讓同一個 agent 可以在多個 model、 keys、甚至 pseudo provider 之間切換，也讓失敗恢復與 cooldown 有固定語意。

Routing Stack

  • Credential Pool：集中管理 endpoint 與多把 keys。
  • Load Balancer：以 ordered instances 決定重試、delay、resume 與 failover。
  • Agent Access Control：決定 agent 能碰哪些 docs、MCP、skills、built-in tools。

為什麼這樣設計

  • 把 provider 細節從 agent profile 抽離，讓 agent 比較像能力配置，而不是固定模型綁定。
  • 失敗紀錄與 cooldown 放在 instance 層，比放在 agent 層更容易追蹤。
  • 後來的 MAGI、tutorial、browser skill runtime 都可以直接吃這套 routing abstraction。
