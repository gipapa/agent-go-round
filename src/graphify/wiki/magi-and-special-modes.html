MAGI and Special Modes

MAGI 是這個專案裡最明顯帶有「產品表演性」的模式：它不是多 agent generic framework，而是把 Melchior / Balthasar / Casper 三個固定角色變成受控的 deliberation system。

兩種模式

  • Magi Vote：三賢人同時表決，各自回 ballot，再做最終裁定。
  • Magi Consensus：三賢人持續交換摘要與理由，直到一致或 deadlock。

設計邊界

  • 三個 agent 名稱固定，且必須預先存在。
  • 三個角色各自綁定固定的 MAGI skill，不吃全域 docs、MCP、built-in tools、其他 user skills。
  • UI 呈現不是普通 chat bubble，而是 EVA 風格的裁決面板。
MAGI 的重點不是讓 agent 自由發揮，而是把角色差異、回合推進、最終裁決，都收束在一個可觀察的前端介面裡。
